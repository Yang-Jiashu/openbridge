## 🎯 快速开始

### 1️⃣ 一键启动服务端

```bash
cd PC_Commander_Product/PC_Server
npm install
npm start
```

看到以下输出表示成功：
```
╔════════════════════════════════════════════════╗
║     🚀 OpenCode Remote Server 已启动          ║
╠════════════════════════════════════════════════╣
║  📡 地址: http://0.0.0.0:8000                ║
║  💻 本地: http://localhost:8000              ║
╚════════════════════════════════════════════════╝
```

### 2️⃣ 获取电脑 IP

```bash
# Windows
ipconfig

# 找到 IPv4 地址，例如：192.168.1.5
```

### 3️⃣ 手机访问

- 打开手机浏览器
- 访问 `http://192.168.1.5:8000`
- 或使用 Uni-app 打包的 APK

## 💻 服务端部署

### 环境要求

- **Node.js**: >= 16.x
- **操作系统**: Windows / macOS / Linux
- **可选**: OpenCode 或 OpenClaw（用于 AI 功能）

### 安装步骤

#### 1. 安装依赖

```bash
cd PC_Commander_Product/PC_Server
npm install
```

#### 2. 配置环境变量（可选）

创建 `.env` 文件：

```env
PORT=8000
OPENCODE_PATH=C:\Users\33455\Desktop\PC\opencode-dev
OPENCLAW_PATH=C:\Users\33455\Desktop\PC\openclaw
```

#### 3. 启动服务

```bash
# 开发模式
npm run dev

# 生产模式
npm start
```

### API 接口

| 接口 | 方法 | 说明 |
|:---|:---:|:---|
| `/api/health` | GET | 健康检查 |
| `/api/chat` | POST | AI 对话 |
| `/api/code/run` | POST | 运行代码 |
| `/api/files` | GET | 文件列表 |
| `/api/files/read` | GET | 读取文件 |
| `/api/files/write` | POST | 写入文件 |
| `/api/command` | POST | 执行命令 |
| `/api/opencode` | POST | OpenCode 命令 |
| `/api/openclaw` | POST | OpenClaw 命令 |

### 与 OpenCode/OpenClaw 集成

如果你的系统中已安装 OpenCode 或 OpenClaw，服务端会自动检测并使用它们。

```javascript
// 自动检测路径
const possiblePaths = [
    'C:\\Users\\33455\\Desktop\\PC\\opencode-dev',
    'C:\\Users\\33455\\Desktop\\PC\\openclaw',
    // ... 其他路径
];
```

## 📱 移动端打包

### 方式一：HBuilderX 云打包（推荐）

#### 1. 下载 HBuilderX

- 官网: https://www.dcloud.io/hbuilderx.html
- 下载 Windows 标准版

#### 2. 导入项目

1. 打开 HBuilderX
2. 文件 → 打开目录
3. 选择 `Mobile_App_Source` 文件夹

#### 3. 配置 manifest.json

```json
{
    "name": "OpenCode Remote",
    "versionName": "1.0.0",
    "versionCode": "100",
    "appid": "uni.UNIOpenCodeRemote2024"
}
```

#### 4. 云打包

1. 点击菜单栏 **发行** → **原生 App-云打包**
2. 选择 **Android**
3. 证书：使用 DCloud 公用证书
4. 点击 **打包**
5. 等待 2-5 分钟，下载 APK

### 方式二：本地打包

需要安装 Android Studio 和 SDK。

```bash
# 生成 Android 项目
npx @dcloudio/uvm@latest alpha

# 在 Android Studio 中打开并构建
```

### 安装到手机

1. 将 APK 传输到手机
2. 点击安装
3. 允许"未知来源"安装

## 🔧 配置说明

### 服务端配置

#### 端口配置

编辑 `PC_Server/.env`：

```env
PORT=8000  # 修改为你想要的端口
```

#### OpenCode 配置

如果 OpenCode 不在自动检测路径中：

```env
OPENCODE_PATH=你的OpenCode路径
```

### 移动端配置

#### 首次使用

1. 打开 App
2. 进入设置页面
3. 输入服务器地址：`http://192.168.1.5:8000`
4. 点击"测试连接"
5. 保存设置

#### 远程访问

使用 Ngrok 实现外网访问：

```bash
# 安装 Ngrok
npm install -g ngrok

# 启动隧道
ngrok http 8000

# 使用生成的 HTTPS 地址
```

## ❓ 常见问题

### Q: 无法连接到服务器？

**排查步骤：**

1. ✅ 检查服务端是否启动（看到绿色启动日志）
2. ✅ 检查防火墙是否允许 8000 端口
3. ✅ 确保手机和电脑在同一 WiFi
4. ✅ 检查 IP 地址是否正确

**测试连接：**

```bash
# 在电脑上
curl http://localhost:8000/api/health

# 在手机上浏览器访问
http://192.168.1.5:8000/api/health
```

### Q: OpenCode/OpenClaw 未找到？

**解决方案：**

1. 检查路径是否正确
2. 在 `.env` 中手动配置路径
3. 确保已正确安装 OpenCode/OpenClaw

### Q: 代码运行失败？

**检查：**

1. 是否安装了对应的运行时（Node.js、Python 等）
2. 代码是否有语法错误
3. 是否有权限执行

### Q: 如何更新 App？

1. 修改版本号（`manifest.json`）
2. 重新云打包
3. 安装新版本（会自动覆盖）

### Q: 支持哪些编程语言？

- ✅ JavaScript / Node.js
- ✅ Python
- ✅ Shell / Bash
- ✅ PowerShell
- ✅ 任何可以通过命令行运行的语言
