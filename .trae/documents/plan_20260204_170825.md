# 架构升级计划：集成 OpenCode 与 PWA 移动端

为了实现手机端 App 体验并集成 OpenCode (类似 Claude Code) 的强大能力，我建议将技术栈从 Python 迁移到 **Node.js**（因为 OpenCode 原生是 Node.js 生态），并采用 **PWA (渐进式 Web 应用)** 方案解决手机端开发问题。

## 1. 核心架构升级 (Node.js + OpenCode SDK)
OpenCode 是基于 Node.js/TypeScript 构建的，为了最完美地集成其能力（代码理解、LSP 支持、Agent 规划），我们将：
*   **后端重构**：新建 `remote-commander-node` 项目。
*   **集成 SDK**：使用官方 `@opencode-ai/sdk` 与本地 OpenCode 服务通信。
*   **API 服务**：使用 `Express` 提供 HTTP 接口，桥接手机端与 OpenCode Agent。

## 2. 手机端方案：PWA (Progressive Web App)
用户希望手机端是“App 或小程序”，**PWA 是目前最方便开发且体验最接近原生 App 的方案**：
*   **无需开发原生 App**：不需要配置复杂的 iOS/Android 开发环境。
*   **即点即用**：手机浏览器访问 PC IP，点击“添加到主屏幕”，即可像 App 一样全屏运行，有独立的图标和启动画面。
*   **开发效率**：直接复用 Web 技术栈，一次开发，全平台通用。

## 3. 实现步骤
1.  **环境准备**：
    *   创建 `package.json`，管理 `opencode-ai`, `express` 等依赖。
2.  **后端实现 (`server.js`)**：
    *   启动 OpenCode Agent 实例。
    *   提供 `/api/chat` 接口，将自然语言转发给 Agent。
    *   处理 Agent 的流式响应（SSE），实现打字机效果。
3.  **前端升级 (`public/`)**：
    *   **UI 优化**：仿照微信/Claude 的移动端聊天界面。
    *   **PWA 支持**：添加 `manifest.json` (配置图标、应用名) 和 meta 标签，使其可被“安装”。
4.  **打包与运行**：
    *   提供 `start.bat` 一键启动脚本。
    *   (可选) 未来可用 `pkg` 将整个 Node.js 项目打包成单文件 `.exe`。

## 4. 预期效果
*   **PC 端**：运行一个黑盒服务，在后台静默处理任务。
*   **手机端**：桌面上有一个“PC 指挥官”图标，点开就是聊天界面，可以说“帮我检查代码”、“跑个测试”、“看看实验日志”，全部由 OpenCode 强力驱动。
